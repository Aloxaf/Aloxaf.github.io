jarvisoj - pwn - level1

(setq org-babel-python-command "python2")

分析

首先 checksec leve1 

[*] '/home/zhonghua/writeup/pwn/jarvisoj-level1/level1'
  Arch:     i386-32-little
  RELRO:    Partial RELRO
  Stack:    No canary found
  NX:       NX disabled
  PIE:      No PIE (0x8048000)
  RWX:      Has RWX segments

NX没有开启, 很可能是ret2shellcode

拖进IDA, 直接F5

跟进vulnerable_function

read函数存在非常明显的溢出

同时可以发现程序运行时会输出buf的地址, 然后使用read函数读取0x100字节内容

那基本思路应该是先获取buf地址(设为p), 然后构造shellcode+p的组合使得函数返回到buf开头执行我们的代码
